INSTALLATION
--------------------------------------------------------------------------------
Step 1: Open terminal and navigate to the frontend directory

   cd frontend

Step 2: Install dependencies

   npm install

   This installs:
   - react 18.2.0          (UI framework)
   - react-dom 18.2.0      (DOM rendering)
   - react-router-dom 6.20 (Client-side routing)
   - react-scripts 5.0.1   (CRA build toolchain - Webpack, Babel, ESLint)
   - recharts 2.10.0       (Chart components)

Step 3: Verify installation

   node_modules/ folder should be created with all packages

================================================================================

RUNNING THE APPLICATION
--------------------------------------------------------------------------------
Development mode (with hot reload):

   npm start
   OR
   npm run dev

   -> Opens at http://localhost:3000
   -> If port 3000 is occupied, it will prompt to use another port (e.g., 3001)
   -> Auto-reloads on file changes

Production build:

   npm run build

   -> Creates optimized build in /build folder
   -> Ready for deployment to S3/CloudFront

ENVIRONMENT CONFIGURATION
--------------------------------------------------------------------------------
No .env file is required for the frontend demo (all data is mocked).

For production deployment (Req 16), the frontend is deployed to:
- AWS S3 bucket (static hosting)
- Served via CloudFront (HTTPS, HTTP/2+3, SPA fallback)
- API calls point to API Gateway endpoint

To configure API base URL for production:
   Create .env file in frontend/:
   REACT_APP_API_URL=https://your-api-gateway-url.amazonaws.com

COMMON ISSUES & TROUBLESHOOTING
--------------------------------------------------------------------------------
1. "react-scripts is not recognized"
   -> Run: npm install
   -> Or: npx react-scripts start

2. Port 3000 already in use
   -> Press Y when prompted to use alternate port
   -> Or set: $env:PORT=3001; npm start (PowerShell)

3. PowerShell execution policy error
   -> Run: Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

4. Module not found errors
   -> Delete node_modules/ and package-lock.json
   -> Run: npm install

5. Blank page after build
   -> Add "homepage": "." to package.json for relative paths
